1. 01. 베이스테이블 변경점
    - 1~2번 ID 변경
    변경 내용: 장비 아이템 합성에 필요한 데이터 값 입력
    id	unit	data	descripton
    1	4	    2000	투입 아이템의 따른 등급 유지 확률 (만분율)
    2	4	    4000	투입 아이템의 따른 최소 기대값과 최대 기대값의 사잇값 (만분율)

2. 07. 스테이지 테이블 변경점
    변경 내용: 
    1. 일반 스테이지에 나올 수 있는 몬스터 그룹을 010.몬스터 그룹 테이블로 이동
    2. 힐 이벤트 아이디 값 및 등장 확률 기입
    3. 스테이지 최소 갯수 제거
    4. 스테이지 나머지값 제거
    5. normal_monster_per, Elite_monster_per, Shop_per, event_per, heal_event_per 타입 변경
    - 해당 파라미터 값의 경우 스테이지 구성시 방에 해당 타입이 나올 확률을 계산하는 파라미터 값이었지만, 금일 논의를 통해 해당 파라미터값에서 등장 확률이 아닌 전체 생성된 방의 비율 형태로 변경

3. 10. 스테이지 그룹 테이블 변경점
    Key_id 파라미터 추가 (중요)
    Group_id 역할 변경
    stage_id 파라미터 추가
    group_per 파라미터 추가

4. 스테이지 내 방 생성 체계 변경
    예시_1스테이지 x번 방에 들어갈 스테이지의 경우
    1. 07번 테이블에서 스테이지 정보를 받아온다.
    2. 07번 테이블에서 _per로 끝나는 5종의 파라미터 값의 비율에 맞게 해당 방에 들어갈 방의 타입을 정한다.
    3. 만약 일반 스테이지가 선택 되었을 경우, 7번 테이블에 normal_monster_group의 아이디를 체크한다.
    4. 10번 테이블에서 해당값과 일치하는 Key_id를 체크한다.
    5. 해당 키값은 중복되어 존재 할 수 있으며, 해당 키값의 다른 group_id 파라미터 값중 1개를 선택한다.
    6. 선택될 확률은 10번 테이블 group_per를 기준으로 한다.
    7. 해당 키값의 맞는 일반 몬스터 그룹을 해당 방에 세팅한다.
